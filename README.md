# 計時同仁評鑑系統

計時同仁「態度 30 分」評鑑網頁 App：同仁匿名互評 → 自動算態度分 → 主管調整 → 查時薪。架構已預留職能 70 分擴充。

- 前端：GitHub Pages 靜態頁（`index.html` 互評、`admin.html` 管理）
- 後端：Google Apps Script Web App（`apps-script/Code.gs`）
- 資料庫：Google Sheet
- 計分唯一來源：`js/scoring.js`（Apps Script 不重算分數）

設計與計畫文件見 `docs/superpowers/`。

---

## 一、建立 Google Sheet

新增一個 Google 試算表，建立 4 個分頁與標題列：

- **設定**：自由排版，但需用「資料 → 命名範圍」建立下列命名範圍。
- **評分紀錄**（標題列）：`時間戳 | 季度 | 受評者 | 效率 | 互動 | 團隊 | 紀律 | 學習 | 出勤 | 備註 | 指紋`
- **主管調整**（標題列）：`季度 | 受評者 | 態度調整 | 態度原因 | 職能調整 | 職能原因 | 時間 | 操作者`
- **結果**（標題列，供日後匯出對照）：`季度 | 受評者 | 態度分 | 態度調整 | 職能分 | 職能調整 | 最終總分 | 對應時薪 | 互評份數`

### 命名範圍（在「設定」分頁框選後命名）

| 命名範圍 | 形狀 | 內容 |
|---|---|---|
| `CFG_quarter` | 1 格 | 例：`2026-Q1` |
| `CFG_passcode` | 1 格 | 主管/管理頁通行碼 |
| `CFG_ratees` | 1 欄多列 | 受評同仁名單（每格一名） |
| `CFG_items` | 6 列 × 7 欄 | `key｜label｜5星｜4星｜3星｜2星｜1星`（6 項態度題） |
| `CFG_wage` | N 列 × 3 欄 | `分下限｜分上限｜時薪` |

> 6 項態度題的 5→1 星描述沿用現有 Google 表單文字（工作效率與品質、與客人互動、團隊合作與溝通、公司規定遵守度與紀律、學習與主動性、出勤與守時）。

## 二、部署 Apps Script

1. 在該試算表選「擴充功能 → Apps Script」。
2. 把 `apps-script/Code.gs` 內容整段貼進去，存檔。
3. 「部署 → 新增部署作業 → 類型：網頁應用程式」。
   - 執行身分：**我自己**
   - 誰可以存取：**所有人**
4. 授權後取得 Web App URL（以 `/exec` 結尾），複製。

> **每次修改 `Code.gs` 後**，要「部署 → 管理部署作業 → 編輯 → 版本：新版本」才會生效。

export const APPS_SCRIPT_URL = 'https://script.google.com/macros/s/XXXX/exec';

## 四、發布到 GitHub Pages

1. 建立 GitHub repo，推送本專案。
2. Settings → Pages → Branch 選 `main`（或部署分支）、根目錄 → 儲存。
3. 互評連結：`https://<帳號>.github.io/<repo>/`
4. 管理連結：`https://<帳號>.github.io/<repo>/admin.html`

---

## 五、測試與驗收

```bash
node --test     # 計分/驗證/指紋單元測試（應全綠）
```

端到端驗收：
- 互評頁匿名填 2～3 份（不同裝置/瀏覽器）→「評分紀錄」累積。
- 管理頁輸入通行碼 → 總覽態度分＝互評平均加總、時薪查表正確；無互評者顯示「資料不足」。
- 主管調整 ±值 → 反映到最終總分；總分落在時薪表外顯示「需人工確認」。
- 防重：同裝置重送被擋、重整顯示「本季已完成」。
